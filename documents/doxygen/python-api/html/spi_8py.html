Detailed Description

Generic SPI bus driver using /dev and ioctl.

General:

Broadly speaking this driver follows the precedents set by the internal Linux API for device drivers that access their devices via I2C.

For maximum flexibility, this driver does not encode the paths to the SPI nodes. This is because it will often be desirable to make such paths a config option, or to use a symbolic link or other alias in lieu of the usual path.
