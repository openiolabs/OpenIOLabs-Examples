Detailed Description

Special functions for managing persistent flags.

Persistent flags may be used to store small amounts of data on the IOI that remains present after a restart, power-cycle or firmware upgrade (but not a re-imaging of the SD card).

Each flag has a name (which should contain alpha-numeric and _ only) and a value which can be a string, integer or floating-point number.

Writing or attempting to read a non-existant flag creates it.

Flags may also be accessed from the server.
