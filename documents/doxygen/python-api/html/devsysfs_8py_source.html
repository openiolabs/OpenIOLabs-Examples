2 # @package devsysfs

4 # @brief Access to /dev and /sys nodes
5 #
6 # General:
7 #
8 # This is the correct API to use for device drivers when accessing these
9 # nodes under /dev or /sys. Other "file" APIs may not be able to access
10 # these nodes in the correct manner.
11 #
12 # There is a class-based interface which holds the device file open while
13 # the driver class instance exists, and alternative free functions
14 # that open the device, act, and close it again. the latter are termed
15 # "monolithic" and begin with mono_.
16 #
17 # Read and write operations work with lists of byte data only (numpy.uint8 or
18 # numpy.int8). The variants containing _block are for intended for block
19 # devices and use pread() and pwrite(). The others use read() and write()
20 # are meant for character devices. This correspondance is not strict
21 # and some devices don't really correspond to block or character idioms.
22 #
23 # The ioctl can accept data structures made up of pointers, arrays and
24 # structs, and converts them to and from a memory-based form for use
25 # with the underlying ioctl operation. Alignment padding is generated
26 # consistent with the C++11 cross-compiler used to build the scriptml
27 # interpreter.
28 #
29 # Python:
30 #
31 # The python version of ioctl uses lists populated with numpy types
32 # such as numpy.uint8 and so on in lieu of structs. Free function
33 # block() is provided to create a C pointer to a list. This should be
34 # used to seperate contiguous blocks of data (where the ioctl's data
35 # structure requirement is to use a pointer). Otherwise, nested lists
36 # are concatenated into a single block, as with C structures.
37 #

39 # All Rights Reserved
40 #
