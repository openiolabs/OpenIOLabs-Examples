4 # @brief Drive GPIOs using the sysfs interface.
5 #
6 # The GPIO pins are specified using strings. This is because some platforms
7 # use a combination of numbers and letters to specify ports, eg ports
8 # named A, B, C and D and pins A0, A1, ... A7, B0, B1, ... and so on.
9 #
10 # Recommended practice is to hard-code these strings. If GPIO ports need
11 # to be indexed by a numerical expression, simply place the GPIO
12 # instances in an list:
13 #
14 # gpios = [ gpio.device("A0"), gpio.device("A1"), gpio.device("A2"), gpio.device("A3") ]
15 # gpio[index].set_level(...)
16 #
17 # This provides robustness against arbitrary re-assignment of GPIOs.
18 #
19 # It is recommended to ensure that gpio.device objects are destructed before
20 # exiting the executable, because if this is not done, the GPIO device will
21 # be left inaccessible (until unexport is performed manually).
22 #
